31  /**
32   * A DB Instance is an isolated database environment running in the cloud. A DB Instance can contain
33   * multiple user-created databases, and can be accessed using the same tools and applications as a
34   * stand-alone database instance.
35   * 
36   * 
37   * <h4>Note</h4>
38   * 
39   * Amazon RDS supports access from any standard SQL api application. Amazon RDS does not allow
40   * direct host access via Telnet, Secure Shell (SSH), or Windows Remote Desktop Connection.
41   * 
42   * 
43   * DB Instances are simple to create and modify with the Amazon RDS command line tools, APIs, or the
44   * AWS Management Console.
45   * 
46   * 
47   * Amazon RDS creates a master user account for your DB Instance as part of the creation process.
48   * This master user has permissions to create databases and to perform create, delete, select,
49   * update and insert operations on tables the master user creates. You must set the master user
50   * password when you create a DB Instance, but you can change it at any time using the Amazon RDS
51   * command line tools, APIs, or the AWS Management Console. You can also change the master user
52   * password and manage users using standard SQL commands.
