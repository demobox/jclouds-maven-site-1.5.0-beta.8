30  /**
31   * Amazon RDS allows you to control access to your DB Instances using DB Security Groups. A DB
32   * Security Group acts like a firewall controlling network access to your DB Instance. By default,
33   * network access is turned off to your DB Instances. If you want your applications to access your
34   * DB Instance you can allow access from specific EC2 security groups or IP ranges. Once ingress is
35   * configured, the same rules apply to all DB Instances associated with that DBSecurityGroup.
36   * 
37   * <h4>Important<h4/>
38   * 
39   * Please ensure you authorize only specific IP ranges or EC2 security groups. We highly discourage
40   * authorizing broad IP ranges (for example, 0.0.0.0/0).
41   * 
42   * Note that you cannot use Amazon RDS DB Security Groups to restrict access to your Amazon EC2
43   * instances. Similarly, you cannot apply an Amazon EC2 security group to your Amazon RDS DB
44   * Instance.
45   * 
46   * 
47   *
